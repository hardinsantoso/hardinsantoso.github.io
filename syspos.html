October 2015 - November 2015

Web based POS System development

Development project / Operation Tool

Introduction

This project sees me working along another intern to help develop a POS system based on PHP for a shoe and fashion related shop. This project have me assist on translating wireframe into translatable design and to have it functioning as static website before being coupled into the backend. I also helped on the integration of front-end and backk-end.



The Detail

This project details are first discussed between the client and the agency, in which I dont have any influence whatsoever but in this matter I follow and listen to what the client, and the agency which I am intered at wants and wishes for. In here, the client wanted a POS system that is clean, able to do just like any other POS system but they want it to be independent and not uses other software and thus they contracted the agency to do so, the agency, which specializes in designing design be it websites or apps or other graphics, also have capabilities to translate that design into functional product in someway. I and my intern collague then tasked to help assist the wireframing, making it into PSD and then making the front-end to ready it for back-end coupling.

The project runs on Laravel framework, and the CSS on some form of sass and boilerplate.

Summary


This project teaches on how to follow instruction, see what the stakeholder wants, work with others and superior, and be a beneficial contributing part of the team. This development is my first big project that I handled, and was quite technically challenging for me in the beginning, with all sort of things and coordinating work, as opposed to the usual working all the way by myself. This means that I also learn how to document my work well, create readable documentation and read others documentation.
